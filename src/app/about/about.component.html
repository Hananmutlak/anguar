Om uppgiften

Arbetsprocess

Jag började med att skapa en ny Angular-applikation med Angular CLI. Sedan strukturerade jag projektet i tre komponenter: home, convertersida och about. För navigering mellan sidorna använde jag Angular Router med RouterLink.

Konverteringsfunktionaliteten implementerade jag med Two-Way Data Binding och FormsModule. Responsiv design löste jag med CSS Flexbox och media queries.

Tekniska val

  • Använde Angular CLI för projektgenerering
  • SCSS för styling med modulär struktur
  • Standalone Components för bättre kodorganisation
  • Git för versionshantering

Utmaningar

  • Hantering av beroenden mellan komponenter
  • Integrera komponenter och identifiera fel.
  • Jag försökte med optimerad deployment till GitHub Pages, men det fungerade inte, så jag publicerade webbplatsen med Netlify istället.

Slutsatser

  • Angular erbjuder en robust struktur för webbutveckling
  • Komponentbaserad arkitektur underlättar återanvändning
  • Typescript ger typsäkerhet och bättre kodkvalitet
